With current air quality in large metropolitan area's being very poor, the concept to monitor the air quality through HVAC systems (Heating, ventilation, and air conditioning in large, medium and small companies is a necessary step forward, for future health concerns of staff members. As some organisations already know, BMS (Building Maintenance Systems) are highly costly and difficult to maintain on a regular basis. With this in mind, we decided to use a relatively inexpensive ESP8266 microcontroller chip that is WiFi enabled (IEEE 802.11 b/g/n) which has enough memory to implement the solution (4 MB external SPI flash, 2 MB usable space and the other 2 MB for Firmware updates). Administrators could also remotely send firmware updates to these controllers (sensor, connectivity and security updates) through a web interface. It also had 32kb sram; 80kb dram and SPIFFS built into the chip itself, which was more than adequate for the solution. We used the Microchip MCP23017 16-bit serial port expander with I2C serial interface that would do the switching on the Analog Multiplexer 4051 for the analogue sensors. The entire program is written in C++ on the EPS8266 microcontroller, which communicates to a low-level gateway socket server (C#) running in the cloud. We developed the low-level sockets gateway server using C# running on Windows 2012 R2, rather than using an 'out-of-box' solution such as signalr or Socket.IO. With the socket server, I wanted as little overhead and as much control as possible. This would then save the data to a Microsoft SQL Server sitting in the https://www.ionos.co.uk cloud. The gateway socket server, was developed to do over 1000’s reads/writes per minute and runs at less than 30MB on the server with a CPU rate of between 0% to 1%. The results are then displayed and managed in real-time on an ASP.Net MVC 5 website.
